F37: 達成基準 3.2.2 の不適合事例 － ラジオボタン、チェックボックス、又はセレクトリストを変更すると、事前の予告なしに新しいウィンドウを開いている

Important Information about Techniques

See Understanding Techniques for WCAG Success Criteria for important information about the usage of these informative techniques and how they relate to the normative WCAG 2.0 success criteria. The Applicability section explains the scope of the technique, and the presence of techniques for a specific technology does not imply that the technology can be used in all situations to create content that meets WCAG 2.0.

適用（対象）

HTML 及び XHTML

これは、次の達成基準に関連するfailure不適合事例である:

  • 達成基準 3.2.2 (On Input)
    • How to Meet 3.2.2 (On Input)
    • 達成基準 3.2.2 (On Input)を理解する

ユーザーエージェント及び支援技術によるサポート

See User Agent Support Notes for F37.

解説

この文書では、ラジオボタン、チェックボックス、又はセレクトリストの項目のステータスを変更することによって新しいウィンドウが開く場合に発生する不適合事例について解説する。要素が選択された時、状況の変化（フォームの送信、新しいページを開く、新しいウィンドウ）を引き起こすinput要素を生成するために、スクリプトを使用することができる。開発者は代わりに送信ボタンを使用するか（G80.html: を参照）、又はそこで起こる動作を明確に示す必要がある。

事例

不適合事例 1

下記の例では、送信ボタンを使うのではなく、ラジオボタンが選択されたときにフォームが処理されるため、達成基準を満たしていない。

参考リソース

この実装方法に関する参考リソースはない。

関連する実装方法

(今のところ、なし。)

検証

チェックポイント

  1. ページの中からフォームを探す。

  2. フォーム・コントロールのラジオボタン、チェックボックス、又はセレクトリストの項目のステータスを変更すると、新しいウィンドウが立ち上がるかどうかを確認する。

  3. ステップ2の結果として生成されたそれぞれの新しいウィンドウが、事前に利用者に警告されているか確認する。

判定基準

3.を満たしていない場合、この不適合条件が適用され、コンテンツは達成基準に不適合となる。
